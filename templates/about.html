About Syncify

Why was Syncify made?
Syncify was created to solve a common problem among Spotify users: combining multiple playlists into one, sorted by the date each song was added. This tool simplifies playlist management and helps users rediscover songs they've added over time across different playlists, as well as to keep all of your favorite songs in one place.
Is this project on GitHub?
Yes! The source code for Syncify is available on GitHub. You can find it here. We welcome contributions and feedback from the community.
How is my data being used?
Syncify only accesses the Spotify data necessary to combine your playlists. We don't store any of your personal information or playlist data on our servers. The app uses Spotify's official API and adheres to their data usage policies.

You can read more about how your data is being used in our privacy policy.
What tech stack was used?
Syncify is built using Python with the Flask web framework for the backend. The frontend is created with HTML, CSS, and JavaScript. We also use the Spotipy library to interact with the Spotify API.
Why might the system time not reflect my local time?
The system time displayed in Syncify reflects the time zone of our servers, which use Coordinated Universal Time (UTC). This ensures consistency across different geographical locations. Your local time may differ from the displayed system time depending on your time zone. The time shown represents when the song was added to the playlist according to Spotify's servers, not necessarily the exact time in your location when you added the song.

Is Syncify affiliated with Spotify?
No, Syncify is not affiliated with or endorsed by Spotify. Syncify is an independent project that uses Spotify's public API to provide this service to users.
